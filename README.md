# Colorization using Optimization

Python-реализация алгоритма раскрашивания изображений, основанного на методе оптимизации из научной работы *Y. Levin, D. Lischinski, Y. Weiss. "Colorization using Optimization"* (SIGGRAPH 2004). Подход базируется на идее, что **яркость (luminance)** изображения содержит достаточно информации для определения границ объектов, а **цветность (chrominance)** может быть восстановлена путем минимизации функции энергии (сглаживания) при наличии пользовательских подсказок (scribbles). Скрипт использует оптимизацию в цветовом пространстве L*a*b*, суперпиксельную сегментацию и разреженные линейные системы для восстановления цветных каналов при сохранении оригинальной яркости.

## Описание

Скрипт принимает черно-белое (или исходное) изображение, симулирует пользовательские цветовые метки (случайным образом выбирая 1% пикселей) и распространяет цвет на остальные области, опираясь на градиенты яркости. 

Алгоритм решает задачу минимизации разброса цвета между соседними пикселями, которые имеют схожую яркость. Это позволяет цвету «растекаться» внутри объектов, но не перетекать через их границы.

## Как это работает

1. **Предобработка:** Изображение уменьшается до заданного размера (для ускорения вычислений) и переводится в цветовое пространство **YUV**.
2. **Генерация подсказок (Scribbles):** Случайным образом выбирается доля пикселей (`SCRIBBLE_RATIO`), их оригинальные значения каналов U и V принимаются за «истину» (граничные условия).
3. **Построение графа:** Создается разреженная матрица (Laplacian matrix), где веса ребер между соседними пикселями (8-связная окрестность) вычисляются по формуле Гаусса на основе разности их яркости (канал Y).
4. **Решение СЛАУ:** С помощью `scipy.sparse.linalg.spsolve` решается система линейных уравнений для каналов U и V.
5. **Реконструкция:** Оригинальный канал Y объединяется с восстановленными U и V, после чего изображение переводится обратно в RGB/BGR.

## Установка и запуск

### Требования
Для работы необходимы следующие Python-библиотеки:
```bash
pip install numpy opencv-python scipy matplotlib
```

### Запуск

1. Поместите ваше тестовое изображение в корень репозитория и назовите его `input.jpg` (или измените путь в коде).
2. Откройте и запустите Jupyter Notebook:

```bash
jupyter notebook main_python.ipynb
```

3. Скрипт сохранит промежуточные и финальные результаты:
   - `bw_output.jpg` — канал яркости (Y).
   - `dirichle_output.jpg` — яркость с наложенными цветовыми метками.
   - `restored_output.jpg` — финальное раскрашенное изображение.


## Настраиваемые параметры
В начале скрипта можно изменить следующие переменные:

| Параметр | Значение по умолчанию | Описание |
| :--- | :--- | :--- |
| `INPUT_IMAGE` | `"input.jpg"` | Путь к исходному изображению. |
| `MAX_DIM` | `300` | Максимальный размер стороны изображения в пикселях. Уменьшение размера значительно ускоряет решение СЛАУ. |
| `SIGMA` | `0.05` | Параметр функции сродства (affinity). Меньшие значения делают границы цвета более строгими (цвет не пересекает перепады яркости). |
| `SCRIBBLE_RATIO` | `0.01` | Доля пикселей (от 0.0 до 1.0), которые будут использованы как цветовые подсказки (scribbles). |

## Результаты работы

В процессе выполнения ноутбук выводит сравнительную визуализацию:

1. **Оригинал** — исходное изображение.
2. **Ч/Б изображение** — канал яркости, на котором алгоритм ищет границы.
3. **Восстановлено** — результат работы алгоритма оптимизации.
